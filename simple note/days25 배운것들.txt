톰캣은 서버, 크롬은 클라이언트!
서버가 열려있고, 서버의 ip 주소만 알고 있다면 desktop이나 스마트폰 등의 기기로 internet 망을 통해 그 페이지를 볼 수 있다.

ip주소는 '123.123.123.123'과 같이 생겼다.
위에서 숫자 123 이 부분을 '옥텟'이라고 하고, 옥텟은 0~255의 숫자를 가진다.

도메인 주소 : www.naver.com 과 같은 것.
클라이언트는 도메인 주소를 DNS Server에 보내고, DNS Server에서는 클라이언트에 이에 해당하는 ip 주소를 보낸다.
-> 받은 ip 주소를 이용해 해당하는 Server에 요청하고 데이터를 받는다.


현재는 IPv6가 만들어졌다.
기존 ip주소는 숫자가 부족해서 문제가 생겼다.
이를 16진수의 4자리 숫자 옥텟 8개로 묶인 주소를 만들었다.
86AB:86AB:86AB:86AB:86AB:86AB:86AB:86AB <- 이런 모습


<title>페이지의 타이틀입니다.</title>

<body>
안녕하세요 반갑습니다...페이지의 바디입니다...
</body>

<title> : 웹 페이지의 이름
<body> : 웹 페이지의 내용을 넣는 태그
<br/> : 줄바꿈 태그(단일태그)

06_Image (실제로 많이 쓰이는 태그)

- <img src="파일경로/파일명.확장자명"/ width="가로크기범위(숫자)" height="높이크기범위(숫자)"> : 이미지를 웹 페이지에 표시하는 태그.(단일태그) src 속성을 통해 '어떤 그림'을 '어떻게' 표시할지 기술한다.
※ width, height 둘 중 하나만 설정하면 사진의 원래 비율대로 나온다.

※ 경로, 파일명, 확장자명을 정확하게 써야한다! 그렇지 않으면 사진이 나오지 않는다!

09_AnchorTag2

- 현재 페이지에서 특정 위치로 이동하는 링크

여러 문단들 있는데 링크를 클릭하면 해당 위치로 이동하는것. 위키같은데에 링크가 걸린 숫자나 문장 누르면 해당 문서 위치로 넘어가는 그것.

이동할 위치에 id 옵션을 가진 태그가 필요하다.( ex - <h1 id="first">)
Anchor 태그의 href 값에 "#"과 같이 써주면 이동이 가능하다.( ex - <a href="#first">이동1</a>, '이동1'이라는 링크를 누르면 id="first"를 지정한 위치로 페이지가 이동된다.
-> 즉, #yes 쓰면 id가 yes인 태그를 찾아간다.
